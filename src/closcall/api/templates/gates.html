How this application was built — 15 gates, one honest pipeline

ClosCall was built gate by gate: each stage had hard exit criteria, produced checkable evidence, and could not be self-waived. This page is the whole machine and the whole journey — from an empty repo to the browser you are using right now.

The pipeline

GATES 2–3

GATE 4

GATE 5

GATE 8

classical vs learned (the finding)
GATES 9, 12.5
Diagnose

GATE 10

GATES 11, 14
Execute
guarded mutation, audit-first, rollback
GATE 11

The loop closes: execution is read back from the fabric, recovery is checked against service predicates, and every step lands in the append-only audit trail.
